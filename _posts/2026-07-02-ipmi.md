## 개요

IPMI는 서버 하드웨어를 OS와 독립적으로 원격 모니터링·제어할 수 있는 표준 인터페이스입니다. 서버가 꺼져 있거나 OS가 응답하지 않는 상태에서도 동작합니다.

랙 토폴로지 프로젝트에서 **물리 서버의 하드웨어 정보를 수집**하는 용도로 활용했습니다.

---

## 핵심 개념

### BMC (Baseboard Management Controller)

IPMI의 실행 주체입니다. 메인보드에 내장된 독립 마이크로컨트롤러로, 자체 전원과 네트워크 인터페이스(IPMI over LAN)를 가집니다.

```
[서버]
 ├── CPU / RAM / Storage   ← OS가 제어
 └── BMC                   ← IPMI가 제어 (OS 독립)
      ├── 전원 제어
      ├── 센서 데이터 (온도, 전압, 팬 속도)
      └── 시스템 이벤트 로그 (SEL)
```

제조사별 BMC 구현체:

| 제조사 | BMC 명칭 |
|--------|---------|
| Dell | iDRAC |
| HP | iLO |
| Supermicro | IPMI (자체) |
| Lenovo | XClarity |

### 채널

IPMI 통신 경로입니다. 일반적으로 두 가지를 사용합니다.

- **IPMI over LAN**: 전용 관리 네트워크를 통한 원격 접근
- **KCS (Keyboard Controller Style)**: OS 내부에서 로컬 접근 (`/dev/ipmi0`)

## ipmitool 주요 명령어

```bash
# 전원 상태 확인
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> power status

# 전원 제어
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> power on
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> power off
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> power reset

# 센서 데이터 조회 (온도, 전압, 팬 속도)
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> sdr list

# 시스템 이벤트 로그 조회
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> sel list

# FRU (Field Replaceable Unit) 정보 — 시리얼 번호, 모델명
ipmitool -I lanplus -H <BMC_IP> -U <USER> -P <PASS> fru print

# 로컬에서 접근할 경우 (-I lanplus 생략)
ipmitool sdr list
ipmitool fru print
```

## 랙 토폴로지에서의 활용

랙에 등록된 물리 서버의 BMC IP로 IPMI 요청을 보내 아래 정보를 수집했습니다.

| 수집 항목 | IPMI 명령 |
|----------|-----------|
| 전원 상태 (on/off) | `power status` |
| CPU 온도 | `sdr list` |
| 팬 속도 | `sdr list` |
| 시리얼 번호 / 모델명 | `fru print` |
| 하드웨어 이상 이벤트 | `sel list` |

공급자 비활성화 시 해당 서버들은 수집 대상에서 자동 제외되도록 DB 쿼리로 필터링했습니다.

## IPMI 버전

| 버전 | 주요 추가 기능 |
|------|--------------|
| IPMI 1.0 | 기본 센서 모니터링, 이벤트 로그 |
| IPMI 1.5 | IPMI over LAN, 원격 전원 제어 |
| IPMI 2.0 | AES 암호화, Serial over LAN (SoL), VLAN 지원 |
