## systemd 관리

```bash
# 서비스 제어
systemctl enable <service>
systemctl start / stop / restart <service>

# 로그 확인
journalctl -u <service> -f
```

실습 방향: unit 파일을 직접 작성하고, 의도적으로 장애를 유발한 후 원인 파악 및 복구 과정을 반복했습니다.

## 디스크 관리

```bash
# 파티션 확인 및 생성
lsblk
fdisk /dev/sdb

# 포맷
mkfs.ext4 /dev/sdb1

# 마운트
mount /dev/sdb1 /mnt/data

# 영구 마운트 (/etc/fstab 등록)
echo "/dev/sdb1 /mnt/data ext4 defaults 0 2" >> /etc/fstab
```

---

## 커널 파라미터 튜닝

```bash
# 실시간 적용
sysctl -w net.ipv4.ip_forward=1

# 영구 적용
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
sysctl -p
```

---

## 로그 분석

```bash
# 시스템 로그
journalctl -xe

# 커널 로그
dmesg | tail -50

# 시간 필터링
journalctl --since "1 hour ago"
```

실무 중심으로 실제 장애를 재현하고 근본 원인을 분석하는 방식으로 학습했습니다.
