## 개요

SNMP는 네트워크 장비(스위치, 라우터 등)의 상태를 원격으로 모니터링하고 관리하기 위한 표준 프로토콜입니다. UDP 161 포트를 사용합니다.

랙 토폴로지 프로젝트에서 **물리 스위치의 포트 연결 정보와 상태를 수집**하는 용도로 활용했습니다.

---

## 핵심 개념

### 구성 요소

```
[Manager]  →  GET / SET 요청  →  [Agent]
   ↑                                 │
   └────────── Trap (이벤트) ─────────┘
```

- **Manager**: SNMP 요청을 보내는 쪽 (모니터링 서버, 수집 서비스)
- **Agent**: 장비 내부에서 실행되는 SNMP 데몬 (스위치, 서버)
- **Trap**: 장비에서 Manager로 비동기적으로 보내는 이벤트 알림

### OID (Object Identifier)

SNMP가 관리하는 모든 값은 OID라는 계층형 숫자 경로로 식별됩니다.

```
1.3.6.1.2.1.2.2.1.8   → 인터페이스 운영 상태 (ifOperStatus)
1.3.6.1.2.1.2.2.1.2   → 인터페이스 이름 (ifDescr)
1.3.6.1.2.1.2.2.1.5   → 인터페이스 속도 (ifSpeed)
```

### MIB (Management Information Base)

OID의 의미를 정의한 데이터베이스입니다. 제조사마다 표준 MIB 외에 전용 MIB를 추가 제공합니다.

- **Standard MIB-II** (`1.3.6.1.2.1`): 인터페이스, IP, TCP/UDP 등 범용
- **Vendor MIB**: Cisco, Juniper 등 제조사별 확장 정보

### Community String

SNMP v1/v2c의 인증 수단입니다. 요청 시 community string이 일치해야 응답합니다.

- `public`: 읽기 전용 (기본값)
- `private`: 읽기/쓰기

---

## SNMP 버전

| 버전 | 보안 | 특징 |
|------|------|------|
| v1 | Community String | 기본, 암호화 없음 |
| v2c | Community String | 벌크 조회(GetBulk) 추가, 성능 향상 |
| v3 | 사용자 인증 + AES 암호화 | 보안 강화, 운영 환경 권장 |

## 주요 SNMP 명령어

```bash
# 단일 OID 조회
snmpget -v2c -c public <SWITCH_IP> 1.3.6.1.2.1.1.1.0

# OID 하위 전체 조회 (트리 탐색)
snmpwalk -v2c -c public <SWITCH_IP> 1.3.6.1.2.1.2.2

# 인터페이스 목록 조회
snmpwalk -v2c -c public <SWITCH_IP> 1.3.6.1.2.1.2.2.1.2   # ifDescr (이름)
snmpwalk -v2c -c public <SWITCH_IP> 1.3.6.1.2.1.2.2.1.8   # ifOperStatus (up/down)
snmpwalk -v2c -c public <SWITCH_IP> 1.3.6.1.2.1.2.2.1.5   # ifSpeed

# 벌크 조회 (v2c 이상, 대량 데이터 효율적 수집)
snmpbulkwalk -v2c -c public <SWITCH_IP> 1.3.6.1.2.1.2.2
```

## 스위치 포트 상태 값

`ifOperStatus` 응답값:

| 값 | 의미 |
|----|------|
| 1 | up |
| 2 | down |
| 3 | testing |

## 랙 토폴로지에서의 활용

랙에 등록된 스위치 IP로 SNMP 요청을 보내 아래 정보를 수집했습니다.

| 수집 항목 | OID |
|----------|-----|
| 포트 이름 | `ifDescr` (1.3.6.1.2.1.2.2.1.2) |
| 포트 운영 상태 (up/down) | `ifOperStatus` (1.3.6.1.2.1.2.2.1.8) |
| 포트 속도 | `ifSpeed` (1.3.6.1.2.1.2.2.1.5) |
| 시스템 이름 | `sysName` (1.3.6.1.2.1.1.5.0) |

수집한 포트 연결 정보로 **서버 ↔ 스위치 간 물리 연결을 시각화**했습니다. 멀티 공급자 환경에서 동일 스위치에 대한 SNMP 요청이 중복 발생하지 않도록, 스위치 단일 등록 후 공유하는 방식으로 설계했습니다.
