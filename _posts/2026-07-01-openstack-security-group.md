## 개요

보안그룹은 **포트(Port)에 적용**되는 개념입니다. 인스턴스에 직접 적용되는 것처럼 보이지만, 실제로는 인스턴스에 연결된 포트를 통해 적용됩니다.

```
Instance
 ├── Port1
 │   ├── SG-A
 │   └── SG-B
 ├── Port2
 │   └── SG-B
 └── Port3
     └── (보안그룹 없음)
```

포트 하나에 여러 보안그룹을 설정할 수 있으며, 보안그룹이 설정되지 않은 포트도 존재할 수 있습니다.

---

## 인스턴스 생성 시나리오별 동작

### 1. 보안그룹 지정 후 생성

인스턴스 생성 과정에서 포트가 자동 생성되며, 지정한 보안그룹이 해당 포트에 적용됩니다.

### 2. 보안그룹 미지정 후 생성

포트가 자동 생성되고, `default` 보안그룹이 적용됩니다.

### 3. 기존 포트를 지정하여 생성

인스턴스 생성 시 `security_groups` 옵션은 무시됩니다. 포트에 이미 적용된 보안그룹이 그대로 유지됩니다.

### 4. 보안그룹 없는 포트를 지정하여 생성

보안그룹이 적용되지 않은 상태로 인스턴스가 생성됩니다.

### 5. 네트워크(자동 생성) + 보안그룹 없는 포트를 함께 지정하여 생성

인스턴스에 포트 2개가 할당됩니다.

```
Instance
 ├── Port1 (선택한 네트워크의 자동 생성 포트 → 지정한 보안그룹 적용)
 └── Port2 (지정한 포트 → 보안그룹 미적용)
```

---

## 인스턴스에 포트 추가

보안그룹이 적용되지 않은 포트도 실행 중인 인스턴스에 추가할 수 있습니다.

이 경우 해당 포트의 보안그룹 설정은 변경되지 않습니다. 인스턴스에 추가한다고 해서 보안그룹이 새로 적용되거나 기존 보안그룹이 덮어쓰이지 않습니다.

---

## 인스턴스 상세 > 보안그룹 표시 방식

인스턴스 상세 화면의 보안그룹 목록은 **인스턴스 자체가 아닌, 연결된 포트에 설정된 보안그룹**을 기반으로 표시됩니다.

Nova API가 인스턴스에 연결된 모든 포트의 보안그룹을 포트별 중복값 포함해서 반환하기 때문에, 동일한 보안그룹이 여러 포트에 설정되어 있으면 중복으로 노출될 수 있습니다.

---

## 인스턴스 포트 기준 보안그룹 조회 API

Nova API의 인스턴스 보안그룹 응답이 포트별 구분 없이 중복값을 포함하는 문제를 해결하기 위해, **포트 단위로 보안그룹을 구분해 반환하는 API를 개발**했습니다. (2026.03)

### 엔드포인트

```
GET /v1/contrabass/admin/compute/servers/{serverId}/ports
```

### 응답 필드

| 필드명 | 자료형 | 설명 |
|--------|--------|------|
| `portId` | String | 포트 ID |
| `portName` | String | 포트 이름 |
| `securityGroupList` | List | 포트에 적용된 보안그룹 목록 |
| `securityGroupList.securityGroupId` | String | 보안그룹 ID |
| `securityGroupList.securityGroupName` | String | 보안그룹 이름 |
| `fixedIps` | List | 포트에 할당된 고정 IP 목록 |
| `fixedIps.ipAddress` | String | IP 주소 |

### 특징

- 인스턴스에 연결된 포트별로 보안그룹을 분리해서 반환
- Nova API와 달리 포트 단위 구조로 중복 없이 조회 가능
- 보안그룹이 없는 포트는 `securityGroupList`가 빈 배열로 반환
